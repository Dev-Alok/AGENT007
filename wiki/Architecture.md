# Architecture

## Technical Stack
- Language: Go 1.22+
- LLM Integrations: Ollama (native JSON RPC), LM Studio (OpenAI-compatible endpoints)
- Paradigm: Concurrency-based Execution Loop

## Core Architecture

The Agentic Coder is split into three main components:

1. **Orchestrator (`pkg/orchestrator/agent.go`)**: 
   The heart of the application. The Agent tracks conversation history, interacts with the LLM via `getLLMResponse()`, and dynamically streams out the AI's step-by-step thoughts. It manages an intelligent execution loop that parses JSON output and invokes the requested tools asynchronously mapped in the registry.

2. **LLM Client (`pkg/llm/client.go`)**:
   Provides an abstraction layer to communicate with both Ollama native models and OpenAI-compatible models (like running `llama-3` in LM Studio). It implements brace-depth tracking logic in `ParseToolCalls` to safely extract natively nested JSON schemas from raw text strings generated by the LLM.

3. **Tool Registry (`pkg/tools/file_tools.go`)**:
   A mapping interface containing the actual implementations of file manipulation, searching (`grep`), web fetching, and shell execution. These tools provide the autonomous agent with complete read/write access to the local user environment.
